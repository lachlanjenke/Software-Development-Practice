What differences did you observe moving from procedural to class-based design?
It is quite useful being able to store and recall the values we input. 
The same pay can now be calculated without having to remember or re-enter values. 
Input validation can be handled by the class itself rather than the program using the function.
Just the hours work or rate can be modified before calculating pay, rather than having to re-enter both.

How does static typing affect your workflow?
I haven't come from a language like python that uses dynamic typing. The static typing of C# is esentially the same
as what I have experienced in languages like C or C++. I imagine coming from Python it would mean you have to be
far more intentional with your variables, managing their types yourself rather than letting the system environment
handle them.
